The Shell

  • Command line tool
  • Different shells on different systems

命令行的第一个参数是命令行调用程序的名称，而后的几个参数根据程序的具体功能进行选择。比如下面的 echo 程序，将后面的参数作为输出流，在没有重定向的情况下直接输出到屏幕上，即标准输出。

注意在使用 echo 输出 hello world 时，应为如果直接输入的话会被分开视作两个参数同样如下有两种解决方法，一种是使用双引号或单引号，将想要输入的内容勾选出来。这样就会是输入一整个字符串，抑或者是在两个中间的空格使用转义符号进行连接以代替原本的空格。

Environment Path

这里的 which 这个工具可以帮助我们找到该程序的具体路径，其实在 Windows 系统下也有类似的方式。当你的程序需要运行一个命令行程序的时候，它需要检索一个路径去去查找你的第一个参数，即你需要调用的程序，比如在使用 VScode 进行环境配置的时候我们需要将对应的编译器，包括 GCC，GDB 去添加到 Windows 系统的环境变量中。之后在 cmd 中通过输入 gcc -v 来查询你是否已经将这个程序加入到默认的路径中，这样 VScode 才能调用这个程序进行一系列诸如编译的操作。

基本的文件与路径操作

mv

将一个文件移动到另一个位置

cp

复制一个文件或路径到其他地方

mkdir

创建一个路径

rm

删除一个文件

rmdir

sudo

有些操作是较为危险的，比如删除文件路径等。这种操作就需要我们有更高的权限进行执行。第一种方法是在执行的命令前加上 sudo，如

第二种方式是进入了新的一个新的具有管理员权限的 shell，在其中执行的所有操作都相当于使用管理员权限进行。不建议长时间使用具有管理员权限的 shell ，这有可能会导致误操作。

输入密码后就可以看见之后的所有操作都是在 root 中进行操作，并且命令行情的字符变成了 # ，想要退出的话就输入 exit 。

stream

流（stream）是计算机程序交互的的一种方式。你可以类比地把它理解成一种函数。它处理传输的字符串，将处理后的字符串进行输出。标准的流输入是从键盘进行的。标准流输出是显示在屏幕上的。使用重定向的功能，我们可以将一个程序处理后的输出流导入到另一个程序的输入流。这样可以实现一种类似于编程的效果。

以上是 Shell 以及 Linux 系统使用命令行的一些方法。这种命令行的操作并不局限于一次只能执行一种命令。我们可以将命令组合起来，使用一种类似于编程的方法，使其形成更加复杂的结构，从而实现更多自动化功能。这种思考方式向下发展就是脚本。而相关内容会在下一章中引出。
